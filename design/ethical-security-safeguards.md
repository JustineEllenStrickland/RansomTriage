# Ethical and Security Safeguards

| Risk Area | Design Safeguard |
|---|---|
| Privacy and confidentiality | Use public, synthetic, or sanitized scenarios and avoid unnecessary personal identifiers. |
| Data minimization | Store only workflow responses, candidate mappings, evidence recommendations, and structured notes needed for evaluation. |
| Rule integrity | Separate ATT&CK mapping files and workflow rules from user generated case notes. |
| Automation bias | Label mappings as possible or candidate findings rather than confirmed incident classifications. |
| Telemetry variation | Ask which evidence sources are available and document unavailable telemetry. |
| Traceability | Use GitHub commits and branches to track changes to requirements, mappings, design files, and tests. |
